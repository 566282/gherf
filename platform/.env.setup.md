# Environment Configuration

## Required Environment Variables

### Supabase Configuration
```bash
VITE_SUPABASE_URL=https://your-project-ref.supabase.co
VITE_SUPABASE_ANON_KEY=your-public-anon-key
```

### Application Environment
```bash
VITE_APP_ENV=development|staging|production
```

## Setup Instructions

1. Create a Supabase project at https://supabase.com
2. Copy your project's URL and anon key to `.env.local`
3. Run database migrations:
   ```bash
   npm run db:migrate
   ```
4. Seed initial data (optional):
   ```bash
   npm run db:seed
   ```

## Development

```bash
cp .env.example .env.local
npm install
npm run dev
```

The app will run at http://localhost:5173

## Production Build

```bash
npm run build
npm run preview
```

## Deployment to Netlify

1. Connect your GitHub repo to Netlify
2. Set environment variables in Netlify dashboard
3. Build command: `npm run build`
4. Publish directory: `dist`
5. Deploy

## Security Notes

- Never commit `.env.local`
- The `VITE_SUPABASE_ANON_KEY` is public; it's your project's read-only key
- All sensitive operations are protected by Supabase RLS policies
- Admin actions require `super_admin` role, enforced at database level
